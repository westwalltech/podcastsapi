# Security Policy

| Version | Supported          |
| ------- | ------------------ |
| 1.x.x   | :white_check_mark: |

## Reporting a Vulnerability

We take security seriously. If you discover a security vulnerability within this package, please follow these steps:

1. **Do not** disclose the vulnerability publicly until it has been addressed.

2. **Email us** at [tech@newsongpeople.com](mailto:tech@newsongpeople.com) with:
   - A description of the vulnerability
   - Steps to reproduce the issue
   - Potential impact of the vulnerability
   - Any suggested fixes (optional)

3. **Response time**: We will acknowledge receipt of your report within 48 hours and provide an estimated timeline for a fix.

4. **Disclosure**: Once the vulnerability is fixed, we will:
   - Release a patched version
   - Credit you in the changelog (unless you prefer to remain anonymous)
   - Publish a security advisory if appropriate

## Security Best Practices

When using this addon:

- Keep the addon updated to the latest version
- Use HTTPS in production environments
- Follow Statamic's security guidelines
- Keep your API keys and credentials secure
- Regularly review your environment configuration
